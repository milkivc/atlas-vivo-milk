# Security Policy

## Supported Versions

We provide security updates for the following versions of the Atlas Vivo datasets:

| Version | Supported |
|---------|-----------|
| v1.0.x  | Yes     |
| < v1.0  | No      |

---

## Reporting a Vulnerability

If you discover a security vulnerability in the Atlas Vivo datasets or infrastructure, do not open a public issue. Instead, please send an email to security@associacaomilk.pt with the following information:

- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)
- Your contact information (optional)

We will acknowledge your report within 48 hours and provide a more detailed response within 72 hours, including our assessment of the vulnerability and a plan for addressing it.

---

## Security Updates

When we become aware of a security vulnerability that affects the Atlas Vivo datasets, we will:

1. Assess the vulnerability and determine its severity
2. Develop a fix or mitigation
3. Test the fix thoroughly
4. Release the fix as soon as possible
5. Notify affected users if necessary

---

## Security Best Practices

### For Contributors

- Never commit sensitive information (API keys, passwords, tokens) to the repository
- Use environment variables or secret management tools for sensitive data
- Keep dependencies updated to their latest secure versions
- Review code changes for potential security issues
- Follow the principle of least privilege

### For Users

- Always use the latest version of our datasets
- Review the data before use to ensure it meets your security requirements
- Report any suspicious activity to security@associacaomilk.pt

---

## Data Protection

The Atlas Vivo project takes data protection seriously. All datasets are:

- Processed in compliance with RGPD (General Data Protection Regulation)
- Anonymized or pseudonymized where personal data is involved
- Stored securely with appropriate access controls
- Retained only for as long as necessary

If you have concerns about data protection, please contact our Data Protection Officer at dpo@associacaomilk.pt.

---

## Incident Response

In the event of a security incident, we will:

1. Contain the incident to prevent further damage
2. Eradicate the root cause of the incident
3. Recover affected systems and data
4. Investigate the incident to understand what happened
5. Document lessons learned and improve our security posture

---

## Disclosure Policy

We follow a responsible disclosure policy. When we discover or are notified of a security vulnerability:

- We will acknowledge receipt of the report
- We will investigate the vulnerability
- We will develop and test a fix
- We will release the fix according to our assessment of the risk
- We will credit the reporter (if they wish to be credited)

---
